Salesforce Developer Edition - Correção de Alterações em Contatos

Trailhead Challenge 1 | Identify data loss and restore data
📋 Objetivo: Este guia apresenta o passo a passo para identificar e reverter alterações indevidas no campo E-mail dos contatos no Salesforce.

🎯Sobre este Desafio Trailhead

Este é o Desafio 1 - Identify data loss and restore data do Salesforce Trailhead, a plataforma oficial de aprendizado da Salesforce. Você irá praticar habilidades reais de administração e desenvolvimento, trabalhando com histórico de registros e código Apex para resolver um problema do mundo real.

⭐ Ao completar, você ganhará pontos e badges para seu perfil Trailblazer!

1
Inicie a Org
Clique no botão "Launch" para abrir o ambiente Salesforce Developer Edition.
📌 Observação: É o ambiente necessário para completar o desafio.
2
Investigue a Alteração
Acesse o aplicativo Sales e vá para a guia Contacts (Contatos).
Clique em qualquer contato para iniciar a investigação.
📌 Observação: Você precisará analisar o histórico do contato para identificar as mudanças.
3
Identifique o Usuário
Na ficha do contato, localize o Contact History (Histórico do Contato) para descobrir quem alterou o campo E-mail.
📌 Resultado: O usuário identificado será o "marketing integration user".
4
Obtenha o ID do Usuário
Use a busca global para encontrar o perfil desse usuário.
Copie o ID do usuário que aparece na URL do navegador.
📌 Importante: Você precisará desse ID para o código Apex no próximo passo.
5
Abra o Console
Vá em Setup (Configuração) e lance o Developer Console (Console do Desenvolvedor).
Abra a janela Execute Anonymous Window em:
Debug > Open Execute Anonymous Window
📌 Observação: Esta é a interface onde você executará o código de correção.
5.1
Código Apex para Correção
Cole o código abaixo na janela Execute Anonymous Window.
⚠️ IMPORTANTE: Substitua 'YOUR-MARKETING USER ID' pelo ID do usuário que você copiou no Passo 4. 
List<Contact> contactsToUpdate = new List<Contact>();
List<Id> contactIds = new List<Id>();
Map<Id, String> contactIdToOldEmail = new Map<Id, String>();

for (ContactHistory historyRecord : [
    SELECT Id, ContactId, OldValue 
    FROM ContactHistory 
    WHERE Field = 'Email' AND CreatedById = 'YOUR-MARKETING USER ID'
]) {
    contactIds.add(historyRecord.ContactId);
    contactIdToOldEmail.put(historyRecord.ContactId, (String) historyRecord.OldValue);
}

for (Contact contactRecord : [
    SELECT Id, Email 
    FROM Contact 
    WHERE Id IN :contactIds
]) {
    contactRecord.Email = contactIdToOldEmail.get(contactRecord.Id);
    contactsToUpdate.add(contactRecord);
}

update contactsToUpdate;
💡 O que o código faz: 1. Busca o histórico de alterações de e-mail feitas pelo usuário específico
2. Recupera os valores antigos dos e-mails
3. Atualiza os contatos com os e-mails originais
6
Prepare e Cole o Código
Insira o script Apex de correção na janela.
Substitua o placeholder pelo ID do usuário que você copiou no Passo 4.
📌 Função: O código irá localizar e reverter os e-mails alterados por esse ID.
7
Execute a Correção
Clique no botão "Execute".
✅ Confirmação: Verifique se a execução retorna o status "success" (sucesso).

✅ Finalizando o Desafio 1

Após executar o código com sucesso:

  1. Retorne à página do desafio no Trailhead
  2. Clique no botão "Verify Step" ou "Check Challenge"
  3. Aguarde a validação automática
  4. Receba seus pontos e continue sua jornada! 🎉

💡 Dica: Se a verificação falhar, revise se você substituiu corretamente o ID do usuário no código e se a execução retornou "success".

🛡️ Desafio 2

Prevent and Protect Against Data Loss

🏔️ Trailhead Challenge 2 | Prevenindo Perda de Dados
📋 Objetivo: Configurar medidas preventivas para proteger dados contra perdas futuras, incluindo backup automático e restrições de permissões.
1
Agendar Exportação de Dados
No Setup, pesquise e acesse "Data Export".
Clique em "Schedule Export" para configurar o backup.
📌 Observação: Esta ação agendará backups automáticos dos seus dados do Salesforce.
2
Definir o Cronograma
Defina uma "End Date" (ex: 15 de agosto) e um "Preferred Start Time" (ex: 12 a.m.).
Clique em "Save" para agendar a recorrência do backup.
📌 Observação: O backup será executado automaticamente de acordo com o cronograma definido.
3
Acessar Conjuntos de Permissão
No Setup, pesquise e acesse "Permission Sets".
Clique no conjunto de permissão "marketing integration user".
📌 Observação: Você irá modificar as permissões do usuário que causou a perda de dados.
4
Navegar até Contatos
Dentro do Conjunto de Permissão, clique em "Object Settings".
Encontre e selecione o objeto "Contacts".
📌 Observação: Você está acessando as configurações específicas do objeto Contatos.
5
Remover Permissão de Edição
Clique em "Edit". Encontre o campo "Email" e desmarque a caixa "Edit Access".
🛡️ Importante: Isso impede que o usuário edite o campo E-mail no futuro, prevenindo nova perda de dados.
6
Salvar as Alterações
Clique em "Save".
✅ Resultado: A restrição de segurança está implementada.

✅ Finalizando o Desafio 2

Após concluir todas as configurações:

  1. Retorne à página do desafio no Trailhead
  2. Clique no botão "Verify Step" ou "Check Challenge"
  3. Aguarde a validação das configurações de segurança
  4. Complete o módulo e ganhe sua badge! 🏆

💡 Resumo: Você configurou backup automático de dados e removeu permissões desnecessárias, criando uma camada dupla de proteção contra perda de dados.
